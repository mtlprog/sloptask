# Часть 1 — Инициализация

## Что должно быть готово после этого этапа

Запускаемое приложение с конфигурацией, подключением к БД и заглушками для всех будущих компонентов.

---

## 1. Конфигурация

Приложение должно принимать параметры окружения:

- Адрес и порт HTTP-сервера
- Строка подключения к PostgreSQL
- Интервал запуска фонового процесса проверки дедлайнов (по умолчанию — 1 минута)

---

## 2. Подключение к базе данных

При старте приложение подключается к PostgreSQL и применяет миграции. Если БД недоступна — приложение не стартует.

---

## 3. HTTP-сервер

Сервер слушает на указанном порту. На этом этапе достаточно одного health-check эндпоинта (`GET /healthz`), который возвращает `200 OK` если приложение работает и БД доступна.

Base URL для всех будущих API-роутов: `/api/v1`.

---

## 4. Фоновый процесс (заглушка)

Должен быть заложен механизм запуска периодической задачи (deadline checker). На этом этапе — пустая заглушка, которая запускается по таймеру и логирует факт запуска.

---

## 5. Graceful shutdown

При получении сигнала завершения приложение:

1. Перестаёт принимать новые HTTP-запросы
2. Дожидается завершения текущих запросов
3. Останавливает фоновый процесс
4. Закрывает соединение с БД
